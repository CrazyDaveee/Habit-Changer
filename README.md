# 🌱 习惯改变器 (Habit Changer)

**用随机化减少决策消耗，科学改正坏习惯。**

这是一个纯前端的单页应用，基于行为心理学中**执行功能外部化**的理念，通过**随机决策**帮助你逐步摆脱坏习惯。  
数据 100% 存储在浏览器本地，**绝对不联网**，保护你的隐私。

---

## 🎯 核心理念

- **随机化决策**：把“是否应该做”的决定交给程序，减少意志力消耗。
- **渐进式衰减曲线**：基于伦敦大学学院的 21 天习惯模型，习惯触发概率随天数逐渐下降。
- **日内多次抽签**：针对高频习惯，每天可多次决策，且每次成功避免后概率进一步降低。
- **替代行为池**：抽到“不做”时，随机推荐一个替代行为，并支持自定义权重分配。
- **虚拟陪伴植物**：每次成功避免都会增加好感度，植物从种子长成果实，给予正向反馈。
- **数据看板**：28 天习惯日历、心情热力图、替代行为分布，帮你回顾历程。

## 📝 使用说明

### 第一步：设定目标
- 输入坏习惯名称、当前频率、目标频率（支持 次/天 或 次/周）和目标天数。
- 设置**每次抽签间隔**（默认15分钟）和**每日最大抽签次数**。
- 智能建议会实时提示你目标是否合理。

### 第二步：选择替代行为
- 从预设分类（身体、感官、认知、奖赏类）中勾选，或自定义添加。
- 拖动滑块调整每个替代行为被选中的概率（总和保持100%）。

### 主界面
- **概率曲线**：展示习惯发生的概率随时间下降的趋势，红点标记你当前的位置。
- **抽签按钮**：点击后随机决定“允许发生”或“避免”，若避免则展示替代行为。
- **心情记录**：抽签后可点击表情记录当前心情，并可选填一句话。
- **虚拟植物**：好感度随成功次数增长，植物会从种子长成果实。

### 数据看板
- **习惯日历**：过去 28 天的决策记录、心情和经验变化一览。
- **替代行为分布**：饼图展示你使用过的替代行为比例。

### 设置（⚙️）
- 修改抽签间隔和每日最大次数，保存后即时生效。
- 导出数据为 `.json` 备份，或导入旧数据恢复。
- **AI 曲线生成**：生成一段 Prompt 复制到 ChatGPT 等 AI 工具，获取个性化概率曲线数据后粘贴回应用。

一、原理内涵

1.1 我们为什么很难改掉坏习惯？

你有没有过这样的经历：明明告诉自己“今天不能再打游戏了”，但手还是不受控制地打开了 Steam？或者下定决心戒掉奶茶，结果路过奶茶店时，不知不觉就飞进去了？

这不是因为你意志力薄弱，而是因为意志力本身就是一种有限的资源。

心理学家 Baumeister 提出的自我损耗理论（Ego Depletion）告诉我们：每次抵抗诱惑都会消耗心理能量，就像手机电量一样，用一点少一点。当你已经用意志力撑过了早自习、数学考试、和同桌的八卦诱惑，到晚上再面对游戏时，电量早就见底了。

那么，有没有办法不靠意志力呢？

1.2 核心原理：执行功能外部化

「习惯改变器」的核心原理叫做执行功能外部化（Externalization of Executive Function）。

听起来很复杂？其实很简单。

执行功能（Executive Function）是大脑的“管理系统”，负责做决策、控制冲动、规划行为。当你纠结“要不要打游戏”时，就是执行功能在工作——它在权衡利弊，在消耗你的心理能量。

外部化的意思是：把这个决策权交出去，交给一个外部系统。

在这个应用中，这个外部系统就是一个随机数生成器。你不需要自己纠结“打还是不打”，你只需要点一下按钮，让概率来决定。如果抽到了“不做”，那就接受结果；如果抽到了“做”，也别自责——这是概率的锅，不是你的错。

这样一来，你的大脑就不需要反复消耗意志力来做决策了。你从一个“挣扎者”变成了一个“执行者”，心理负担大大减轻。

1.3 为什么是随机化？

随机化在这里有三个关键作用：

1. 减轻决策负担：二选一变成一键抽签，不用纠结。
2. 避免完美主义陷阱：如果要求自己“完全戒断”，一旦失败就容易放弃。随机化允许偶尔的波动，更符合人性。
3. 渐进式改变：概率会随着天数逐渐下降，让你在不知不觉中减少习惯。

1.4 配套机制：替代行为与虚拟奖励

光说“不要做”是不够的，你得告诉大脑“那要做什么”。

这就是替代行为池的作用。当抽到“不做”时，系统会随机推荐一个替代行为（比如“深蹲10次”“嚼口香糖”“背5个单词”），帮你把注意力从坏习惯上转移走。

同时，虚拟植物系统提供正向反馈。每次成功避免，植物都会获得经验值，从种子慢慢长成果实。这种即时奖励能强化你的坚持行为。

二、科学依据与分析

2.1 习惯形成的渐进线模型

伦敦大学学院的 Phillippa Lally 在 2009 年发表了一项著名研究，追踪了 96 名志愿者在 12 周内养成一个新习惯的过程。她发现：

- 习惯形成并非一蹴而就，而是遵循渐近线模型：最初几天变化最快，随后逐渐趋于平稳。
- 平均需要 66 天 才能让一个行为变得“自动化”，但个体差异很大（18-254 天）。
- 偶尔漏做一天并不会显著影响整体进程。

基于这个模型，我们的应用使用了负指数衰减曲线来描述习惯发生概率的变化：

    P(t) = P₀ × e^(-k×t) + P_target

- P₀ 是初始概率（根据当前频率计算，比如每天打游戏 3 小时，概率接近 1）
- P_target 是目标概率（你希望降低到的水平）
- k 是衰减速率常数（由目标总天数反推）
- t 是当前天数



2.2 日内多次决策与动态概率

Lally 的模型描述的是“日与日之间”的变化，但对于高频习惯（比如一天打游戏好几次），我们还需要“日内”的控制。

因此，本应用设计了日内多次抽签机制：

- 每次抽签后，概率会根据结果动态调整
- 如果上一次成功避免了，本次概率 ×0.85（信心增强，习惯力量减弱）
- 如果上一次允许发生了，本次概率 ×0.95（监控本身也有抑制效果）
- 设有下限保护（不低于基准概率的 30%），防止概率过低失去随机性

这个设计融合了 Bandura 的自我效能理论——成功经验增强信心，进而降低冲动强度。

2.3 替代行为的心理学依据

行为主义心理学认为，要消除一个行为，最好的办法不是“压抑”，而是用另一个行为替代。这就是“不相容行为”策略。

比如，咬指甲的人可以嚼口香糖——你的嘴巴忙着嚼东西，就没空咬指甲了。打游戏的人可以做几个深蹲——你的身体动起来了，游戏手柄就放下了。

我们的应用将替代行为分为四类：

- 身体类（深蹲、拉伸）
- 感官类（嚼口香糖、闻薄荷）
- 认知类（背单词、做算术）
- 奖赏类（看风景图、吃黑巧克力）

并且允许用户自定义权重，让真正有效的策略更容易被选中。

2.4 虚拟陪伴与情感设计

为什么很多人用健身 App 会坚持不下来？因为数字是冰冷的。

我们设计了一棵虚拟植物，它会随着你的成功次数成长：种子 → 嫩芽 → 小苗 → 开花 → 结果。

这个设计参考了“虚拟宠物”效应——当人们对一个虚拟生命产生情感联结时，会更有动力去“照顾”它。同时，每次抽签后植物都会说一句鼓励的话，即使失败了也不会责备你。

三、创作或表达过程

3.1 灵感来源

这个项目的想法来自我的亲身经历。我是一个高三学生，最大的坏习惯是刷短视频——明明知道该复习了，但手指就是会不自觉地打开抖音。我试过设置屏幕使用时间限制、卸载 App、把手机交给父母……但每次都坚持不了几天。

后来我在心理学选修课上了解到“执行功能外部化”这个概念，突然想到：如果我把“要不要刷”的决定交给一个随机程序，是不是就不用消耗意志力了？

3.2 技术选型

我决定做一个纯前端的单页应用（SPA），原因有三：

1. 隐私保护：所有数据存在用户自己的浏览器里，绝对不上传服务器。
2. 无需注册：打开即用，降低使用门槛。
3. 技术能力匹配：作为高三学生，我的前端技能（HTML/CSS/JS）可以独立完成这个项目。

技术栈如下：

- HTML5 + CSS3（手写现代 UI，支持深色模式）
- Vanilla JS（ES6 模块化，不依赖任何框架）
- Chart.js（概率曲线和数据看板图表）
- localStorage（数据持久化）

3.3 开发过程（分阶段）

Phase 1：设置向导与概率曲线引擎（核心 MVP）

这是整个应用的基石。用户输入坏习惯名称、当前频率、目标频率和目标天数，程序自动生成一条概率衰减曲线。

关键算法：

- 频率 → 概率的映射：使用泊松分布公式 1 - e^(-λ)
- 衰减速率常数 k：使得在目标天数时，衰减差距只剩下 5%
- Chart.js 绘制曲线，并在当前天位置打上红点标记

Phase 2：替代行为池与动态权重

设计了 8 种预设替代行为（四类各两种），并实现了等比例权重再分配算法。

当用户拖动一个滑块的权重变为 a% 时，其余行为按原比例分配剩余权重：

    B_new = (1 - a) × B_old / (B_old + C_old + D_old)

这个算法确保了所有权重之和始终等于 100%，而且用户调整某个行为时，其他行为的相对比例保持不变。

Phase 3：主界面与抽签机制

主界面包含：

- 顶部进度条
- Chart.js 概率曲线图
- 巨大的“抽取今日决定”按钮

点击按钮后会播放 2 秒的符号滚动动画，模拟老虎机效果。根据今日概率和随机数判定结果，如果抽到“避免”，则通过轮盘赌算法从替代行为池中抽取一个推荐给用户。

Phase 4 & 5：数据看板、心情记录与虚拟植物

数据看板包含：

- 28 天习惯日历（类似 GitHub 贡献图的色块）
- 替代行为使用分布饼图

虚拟植物好感度规则：

- 每次成功避免 +10 经验
- 每次允许发生 +3 经验
- 连续成功天数 ×2 额外加成
- 五个阶段：种子(0-50) → 发芽(51-150) → 幼苗(151-300) → 开花(301-500) → 结果(500+)

Phase 6：AI 辅助与数据管理

提供了“AI 曲线生成”功能：程序自动生成一段 Prompt，用户可以复制到 ChatGPT 等 AI 工具获取自定义概率曲线数据，然后粘贴回应用中。

同时支持数据导出为 .json 文件，以及从备份文件恢复。

3.4 调试与优化

在开发过程中，我们遇到了不少问题：

1. 手机端 UI 溢出：日历网格在窄屏上七个格子挤在一起，后来改成了 auto-fill + minmax 自适应列数。
2. 日期时区陷阱：new Date('2026-06-19') 被解析为 UTC 时间，导致在中国时区下日期偏移。最终改为手动解析本地日期。
3. 模块循环依赖：app.js 和 interaction.js 互相导入导致加载失败，通过移除不必要的导出解决了。
4. 好感度异常跳动：调试模式下同一天多次抽签导致经验重复累加。改为每次操作后重新计算全部经验，确保数据一致。

为了验证长期运行效果，我们加入了调试模式（URL 参数 ?debug=true），可以虚拟前进天数、重置当天抽签，方便快速测试多天的逻辑。

四、归纳总结

4.1 创作收获

这个项目是我第一次将一个心理学理论完整地转化为一个可以使用的工具。在这个过程中，我学会了：

1. 把理论落地：“执行功能外部化”听起来很抽象，但要把它变成一个按钮、一条曲线、一个随机数，需要反复思考和测试。
2. 用户体验设计：不只是功能实现，还要考虑“用户会不会用”“会不会坚持用”。比如破戒按钮最初设计为“惩罚”，后来改为温暖的鼓励；心情记录从强制弹窗改为轻量快捷栏。
3. 迭代思维：最初设计是一天一次决策，后来通过和讨论，发现对于高频习惯这不合理，于是改为日内多次决策 + 动态概率。好产品是改出来的。

4.2 对心理学的理解

通过这个项目，我对行为心理学有了更深的理解：

- 意志力不是一种“品德”，而是一种“资源”。就像你不能要求一部手机永远不充电，你也不应该责备自己为什么“不够自律”。
- 环境设计比意志力更重要。把你的目标变成一个自动运行的系统，你就从“对抗诱惑”变成了“跟随系统”。
- 偶尔的失败不可怕。Lally 的研究表明，偶尔漏掉一天不会影响习惯形成的整体进程。重要的是“持续性”，而非“完美性”。

4.3 自我成长

作为一个高三学生，做这个项目的过程也帮助了我自己。

在开发的两个月里，我利用这个工具来减少自己的短视频时间。现在我已经连续 40 多天没有在复习时间刷短视频了——当然，偶尔也会“允许发生”，但我学会了不自责，而是点一下抽签按钮，继续执行。

编程和心理学结合，让我既能“造轮子”，也能“用好轮子”。这是我高中阶段最骄傲的一个作品。

算法全流程详解：从目标设定到每日决策

这部分将完整解释「习惯改变器」背后的数学模型，并说明每个公式为什么这样设计。

---

1. 输入：用户设定的三个核心参数

用户在设置向导中填写：

- 当前频率 F_cur（次/天）：现在每天大约发生多少次坏习惯。若用户选择每周次数，内部会除以7转为每日。
- 目标频率 F_goal（次/天）：希望通过计划达到的每天次数。
- 目标天数 T：计划用多少天来达成这个变化。

这些数值是全部计算的基础。

---

2. 映射：从频率到单次冲动概率

现实情况是：每天发生多少次是结果，而每一次面对诱惑时你有多大概率去做，是内在的冲动强度。

我们需要一个函数，把“每天发生次数”翻译成“每次冲动时的发生概率”。经过比较，我们选用了这个公式：

P = \min\left(0.95,\ 1 - \frac{1}{F + 1}\right)

其中 F 是每日频率。这个函数有以下几个特点：

- F=0 时 P=0，表示从不做的习惯冲动概率为0。
- F=1 时 P=0.5，表示一天发生一次的习惯，每次冲动大约有一半的概率会去做。
- F 很大时 P 趋近于1（但不超过0.95），符合习惯越重越难抵抗的直觉，同时又保留了改善空间。

为什么不用线性比例？  

因为习惯强度存在边际递减效应：从0次到1次的变化感觉很大，从10次到11次感觉不明显。上面的公式正好反映了这种非线性，使得低频区概率增长快，高频区增长慢。

---

3. 核心曲线：负指数衰减模型

我们假设：通过有意识的监控和坚持，习惯的冲动强度会随着天数逐步下降，并且下降速度由快到慢，最终趋于一个稳定水平。这可以用负指数衰减来描述。

设 t 为第几天，曲线公式为：

P(t) = P_0 \times e^{-k \cdot t} + P_{\text{target}}

其中：

- P₀：初始单次冲动概率，由当前频率 F_cur 通过上面的映射公式算出。
- P_target：曲线最终趋近的渐近线值，我们设定为 目标频率与当前频率的比值：
  P_{\text{target}} = \min\left(0.8,\ \frac{F_{\text{goal}}}{F_{\text{cur}}}\right)
  这样设计是为了让图表上的“预测每日次数”正好从 F_cur 下降到 F_goal（后面会详述）。
- k：衰减速率常数，根据目标天数 T 算出，使得第 T 天时剩余的衰减空间只残存5%，确保曲线平滑收敛：
  k = -\frac{\ln(0.05)}{T}

这条曲线是每天第一次抽签时的基准概率，它只与天数有关，每天相同。

---

4. 图表显示：从概率转换到“次/天”

用户并不关心“冲动概率”这个抽象数字，他们更想知道：按照这个趋势，我现在大概一天会发生几次？

于是我们将每天的概率乘以初始当前频率，得到图表 Y 轴上的数值：

\text{显示次数} = P(t) \times F_{\text{cur}}

- 在 t=0（第一天），显示次数 ≈ P₀ × F_cur ≈ F_cur，即当前频率。
- 在 t=T（最后一天），显示次数 = P_target × F_cur = (F_goal/F_cur) × F_cur = F_goal，正好等于用户设定的目标频率。

这样，图表直观展示了从“当前频率”平滑下降到“目标频率”的过程。

---

5. 日内多次决策：动态概率调整

上面的曲线只能告诉“今天整体冲动有多强”，但对于每天可能面临多次诱惑的高频习惯，我们需要在一天之内进行多次随机决策，并且让概率随着每次结果动态变化。

第1次抽签：  

使用当天基准概率 P_base = P(t)。

第 n 次抽签（n>1）：  

根据上一次的结果调整概率：

- 若上一次是“成功避免”→ 本次概率 = 上一次概率 × 0.85  
  （成功抵抗让信心增强，下次冲动强度降低）
- 若上一次是“允许发生”→ 本次概率 = 上一次概率 × 0.95  
  （即使发生，监控本身也对习惯有轻微抑制）

同时设置下限保护：概率不低于 P_base × 0.3，避免连续成功后概率过低失去随机性。

实际决策：  

生成一个0到1的随机数 rand，如果 rand < 当前概率，则判为“允许发生”；否则判为“避免”，并随机抽取一个替代行为推荐给用户。

---

6. 完整流程示例

假设用户设定：当前每天打游戏 10 次，目标每天 2 次，计划用 90 天。

1. 计算初始参数  
   - F_cur = 10, F_goal = 2, T=90  
   - P₀ = 1 - 1/(10+1) ≈ 0.909（每次冲动发生概率约91%）  
   - P_target = 2/10 = 0.2  
   - k = -ln(0.05)/90 ≈ 0.0333
2. 生成曲线  
   第1天基准概率 P(1) ≈ 0.909×e^(-0.0333) + 0.2 ≈ 0.879 + 0.2 ≈ 1.079 → 被限制为1，实际上因为指数衰减很慢，前几天的概率接近初始值。  
   第30天 P(30) ≈ 0.909×e^(-1) + 0.2 ≈ 0.334 + 0.2 = 0.534  
   第90天 P(90) ≈ 0.909×0.05 + 0.2 ≈ 0.245（渐近于0.2）
3. 图表显示  
   将每天的概率乘以 F_cur = 10：  
   - 第1天显示 10×1 = 10次/天  
   - 第30天显示 10×0.534 = 5.3次/天  
   - 第90天显示 10×0.245 = 2.45次/天 ≈ 2次/天
4. 某日实际抽签  
   假设第30天，基准概率0.534。当天第一次抽签用此概率，结果为“允许发生”。第二次抽签概率 = 0.534 × 0.95 = 0.507。若结果为“避免”，第三次概率 = 0.507 × 0.85 = 0.431。以此类推。

---

7. 为什么这样设计？

- 频率 ↔ 概率的双向映射保证了用户输入和图表显示的一致，让抽象的概率变得可理解。
- 负指数曲线符合习惯形成研究中“变化由快变慢”的规律，同时数学上易于计算。
- 日内动态调整体现了自我效能理论：成功使信心增强，概率下降；偶尔的放纵不会带来惩罚，保持正向激励。
- 整体系统把“改变习惯”这个复杂心理过程转化为一套透明、可执行的随机决策机制，真正做到了执行功能外部化。

以上就是本应用的全部算法逻辑。它既非黑箱，也非魔术，而是基于心理学原理的简单数学模型，帮助你一步步走出习惯的引力。
